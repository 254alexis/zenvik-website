# Zenvik Technologies Services Structure

## Scope
This document controls how services are presented on the main-site. The WHMCS portal should not become the service catalog or primary hosting sales experience.

## Presentation Rules
- Keep the main-site broad and discovery-focused.
- Give each major service area its own clear landing path when needed.
- Use pricing pages for package detail.
- Use the portal only for checkout, client accounts, invoices, support, tickets, and customer care.
- Do not invent clients, case studies, prices, or guarantees that are not approved.
